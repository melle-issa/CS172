Awesome exploration! The main implementation aspect that I'd like to revise is that of using a separate post type. This is a great opportunity to coalesce on how we are storing user reusable content (see #34352 and #41717).

So I'd love to see if we can incorporate into the exploration the idea of using wp_block and discriminate behavior based on "sync" status:

  • Full sync — current reusable block implementation
  • Partial sync — the wp:pattern idea The wp:pattern block #48458, orthogonally related to "locking"
  • Unsynced — how patterns work today, essentially once inserted it's a copy

Sync status can either be a meta value of the post type or an aspect of the insertion (full sync and partial sync would exist as wp:block or wp:pattern containers).
